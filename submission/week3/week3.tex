\section{Project Title}
\noindent CVaR-MPPI for Highway Lane-Change Planning.

\section{Main Idea}
A risk-sensitive lane-change planner that augments Model Predictive Path Integral (MPPI) with a Conditional Value-at-Risk (CVaR) objective, replacing the risk-neutral expectation over rollouts with a sampled tail-risk backup so the controller suppresses lane-changes that are efficient on average but unsafe under rare traffic outcomes.

\section{Problem Formulation}
Let $x_k\!\in\!\mathbb{R}^{n_x}$ be the joint ego--traffic state, $u_k\!=\!v_k\!+\!\epsilon_k\!\in\!\mathbb{R}^{n_u}$ the perturbed ego control with $\epsilon_k\!\sim\!\mathcal{N}(0,\Sigma_\epsilon)$, and $w_k$ exogenous traffic noise. The simulated stochastic dynamics $\tilde{x}_{k+1}\!=\!\tilde{F}(\tilde{x}_k,u_k,w_k)$ form an unbiased estimate of the nominal $x_{k+1}\!=\!F(x_k,u_k)$~\cite{ramppi}. Over a horizon $K$ with state cost $q$, terminal cost $\phi$, and risk cost $L(\tilde{x})\!=\!\sum_{k=0}^{K-1}\ell(\tilde{x}_k)$, we seek
\begin{equation}\label{eq:obj}
\min_{v}\,\mathbb{E}\!\left[\phi(x_K)\!+\!\sum_{k=0}^{K-1}\!\Big(q(x_k)\!+\!\tfrac{\lambda}{2}v_k^{\!\top}\Sigma_\epsilon^{-1}v_k\Big)\right]\;\;\text{s.t.}\;\;\mathrm{CVaR}_\alpha\!(L(\tilde{x}))\!\le\!C_u,
\end{equation}
where $\mathrm{CVaR}_\alpha(L)\!=\!\mathbb{E}[L\,|\,L\!\ge\!\mathrm{VaR}_\alpha(L)]$ is the conditional expectation of the worst $1\!-\!\alpha$ tail~\cite{rockafellar}, $\alpha\!\in\!(0,1)$ the confidence level, $\lambda\!>\!0$ the inverse temperature, and $C_u$ caps the admissible tail risk.

\section{Sketch of Method}
\subsection{Offline Component}
The offline stage instantiates the dynamic-programming foundation of CVaR control by Chow et al.~\cite{chow_cvarmdp}. Augmenting the state with a confidence variable $y\!\in\!(0,1]$, the CVaR-MDP value $V^{*}(x,y)\!=\!\min_{\mu}\mathrm{CVaR}_y\!\big(\sum_{t}\gamma^{t}Z_t\,|\,x_0\!=\!x,\mu\big)$ admits a $\gamma$-contractive, concavity-preserving Bellman recursion $\mathbf{T}[V](x,y)\!=\!\min_a[c(x,a)\!+\!\gamma\max_{\xi\in\mathcal{U}_{\mathrm{CVaR}}(y,P)}\sum_{x'}\xi(x')V(x',y\xi(x'))P(x'|x,a)]$~\cite{chow_cvarmdp}. We discretise a coarse three-lane MDP over (lane index, gap-to-lead, ego speed) on $\sim\!10^{3}$ cells with $21$ log-spaced confidence points and run interpolated value iteration (Algorithm~1 of \cite{chow_cvarmdp}) to obtain $V^{*}(x,y)$, which is loaded as the MPPI terminal cost $\phi(x_K)\!\leftarrow\!V^{*}(x_K,\alpha)$ to provide a tail-risk--shaped cost-to-go beyond the rollout horizon. Offline complexity is $\mathcal{O}(|\mathcal{X}|\,|\mathcal{Y}|\,|\mathcal{A}|\,n_{\mathrm{iter}})$, paid once before deployment; runtime queries are linear-interpolation lookups in microseconds.

\subsection{Online Component}
The online stage executes a Risk-Aware MPPI receding-horizon controller~\cite{mppi,ramppi}. At each step we sample $M$ control sequences $u_k^{m}\!=\!v_k\!+\!\epsilon_k^{m}$, propagate $K$-step rollouts under the nominal bicycle to accumulate $S_m$, and for each $m$ simulate $N$ stochastic traffic futures via $\tilde F$ to compute the Monte-Carlo CVaR estimate
\begin{equation}\label{eq:cvar_mc}
\widehat{\mathrm{CVaR}}_\alpha^{m}=\tfrac{1}{N_o}\!\sum_{n=0}^{N-1}\!\mathbf{1}_{\{L(\tilde{x}^{m,n})\ge\widehat{\mathrm{VaR}}_\alpha^{m}\}}L(\tilde{x}^{m,n}),
\end{equation}
the Rockafellar--Uryasev sample estimator~\cite{rockafellar,ramppi}. The penalty is folded into a modified rollout cost $\tilde{S}_m\!=\!A\widehat{\mathrm{CVaR}}_\alpha^{m}\mathbf{1}_{\{\widehat{\mathrm{CVaR}}_\alpha^{m}>C_u\}}\!+\!S_m$, importance weights $\tilde{\omega}_m\!=\!\exp(-(\tilde{S}_m\!-\!\min_m\tilde{S}_m)/\lambda)$ are formed, and the mean control is updated as $v^{+}\!=\!\sum_m\tilde{\omega}_m u^{m}/\sum_m\tilde{\omega}_m$; the first input $v^{+}_0$ is applied and $v\!\leftarrow\!v^{+}$ warm-starts the next step. This stage is suitable because GPU-parallel rollouts handle the continuous nonlinear bicycle within the $\sim\!100$\,ms control budget, and Monte-Carlo CVaR accommodates arbitrary, non-Gaussian traffic disturbances. It complements the offline stage by injecting fine-grained interaction with current surrounding vehicles that the coarse $V^{*}$ cannot resolve, while $V^{*}$ supplies the long-horizon CVaR-Bellman backup that a finite-$K$ rollout would otherwise miss.

\section{Working Example Modeling}
\subsection{Modeling \& Simulation}
The ego vehicle uses a 4-state kinematic bicycle, $\dot{X}\!=\!v\cos\psi$, $\dot{Y}\!=\!v\sin\psi$, $\dot\psi\!=\!\tfrac{v}{L}\tan\delta$, $\dot v\!=\!a$, with controls $u\!=\!(a,\delta)$ and wheelbase $L$. The environment is a three-lane straight highway populated by $4$--$10$ surrounding vehicles whose longitudinal motion follows a stochastic Intelligent Driver Model with Gaussian acceleration noise $w_k\!\sim\!\mathcal{N}(0,\sigma^{2})$ and a $2\%$ random lane-change probability per step, providing the rare-but-significant tail events that motivate a CVaR objective. The state cost $q(x)$ combines a smooth collision potential, a lane-tracking term, a desired-speed term, and an effort penalty; the running risk cost $\ell(\tilde{x}_k)$ is a time-to-collision-shaped potential plus a lateral-encroachment indicator, so high-CVaR rollouts coincide with the near-miss outcomes the planner must avoid.

\subsection{Plan for Implementation}
\begin{enumerate}
\item \textbf{Simulation environment.} The three-lane highway simulator is built by subclassing \texttt{HighwayEnv} from the Farama Foundation \texttt{highway-env} package~\cite{highwayenv} and registered as \texttt{CVaRLaneChange-v0}. The package's built-in road network, kinematic-bicycle ego model with continuous controls $u\!=\!(a,\delta)$, and IDM/MOBIL traffic provide a validated baseline; we override the default IDM longitudinal controller to inject Gaussian acceleration noise $w_k\!\sim\!\mathcal{N}(0,\sigma^{2})$ and add a $2\%$ per-step random lane-change probability for surrounding vehicles, producing the rare-but-significant tail events that motivate the CVaR objective.
\item \textbf{Online controller.} Implement RA-MPPI in Python using JAX \texttt{vmap}/\texttt{jit} for parallel GPU rollouts; target $M\!=\!1024$ control samples, $N\!=\!64$ disturbance futures per sample, $K\!=\!30$, $\Delta t\!=\!50$\,ms.
\item \textbf{Offline solver.} Implement the interpolated CVaR-Bellman operator of~\cite{chow_cvarmdp} in NumPy on a coarse $\sim\!10^{3}$-cell three-lane MDP with $21$ log-spaced confidence points; cache $V^{*}(x,y)$ to disk and query at runtime by linear interpolation.
\item \textbf{Baselines and ablations.} Compare vanilla MPPI~\cite{mppi}, RA-MPPI~\cite{ramppi}, and the proposed CVaR-MPPI (with offline $V^{*}$); ablate over $\alpha\!\in\![0.7,0.99]$, $C_u$, the CVaR penalty weight $A$, and the variance scaling $B$.
\item \textbf{Scenario suite.} Sweep low/medium/high traffic density and inject deterministic near-miss scripts (hard-brake of the lead vehicle, aggressive cut-in into the target lane).
\item \textbf{Metrics.} Collision rate, near-miss rate (TTC$<\!1.5$\,s), lane-change completion time, control smoothness ($\|\Delta u\|^{2}$), and per-step compute latency, averaged over $\ge\!200$ episodes per condition with fixed seeds.
\item \textbf{Reproducibility.} \texttt{highway-env} is pinned as a git submodule with an editable install; offline value table, hyperparameters, and random seeds are versioned alongside the controller code.
\end{enumerate}
